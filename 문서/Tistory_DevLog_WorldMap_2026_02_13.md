# [말머리] 안티그래비티 + 코덱스 오케스트레이션으로 플랜 수립, 코덱스로 코딩 진행

## 오늘 작업 요약
오늘은 월드맵 시스템 개발을 진행했다. 작업 방식은 다음과 같다.

- 안티그래비티와 코덱스로 오케스트레이션하여 구현 플랜을 먼저 수립
- 수립된 플랜을 기준으로 코덱스에서 실제 코드 구현 진행
- 컴파일 에러를 즉시 수정하고 재컴파일로 안정성 확인

## 진행 내용
### 1) 설계/플랜 단계
- 기존 설계 문서를 검토하고 월드맵 기능 범위를 재정의
- 단계별 구현 로드맵(데이터/노드/UI/진입 흐름) 정리
- 구현 우선순위를 기준으로 작업 순서 확정

### 2) 구현 단계
- 월드맵 노드 및 UI 연동 코드 정리
- 파티/영웅 선택 UI 갱신 로직 안정화
- 빌드 에러 수정
  - `StageNode` 관련 타입 비교 오류(CS0019) 대응
  - `HeroSelectionUI` 지역 변수 미할당 오류(CS0165) 대응

### 3) 검증 단계
- Unity 재컴파일 수행
- 기존 3건의 컴파일 에러가 모두 해소됨을 확인

## 트러블슈팅 메모
- 값 형식/기본값 비교 이슈는 `default` 비교 기반으로 안전하게 처리
- `TryGetValue` 사용 시 지역 변수 초기화 경로를 명확히 하여 컴파일러 경고/오류를 제거

## 회고
이번 작업의 핵심은 “설계 먼저, 구현은 짧은 피드백 루프로”였다.
오케스트레이션으로 계획을 고정한 뒤 코덱스에서 구현을 밀어붙이니, 수정 범위와 검증 포인트가 명확해져 전체 속도와 안정성이 좋아졌다.
